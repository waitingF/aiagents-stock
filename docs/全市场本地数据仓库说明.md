# 全市场本地数据仓库说明

本项目通过 git submodule 引入 `stock-data-store`，用于维护 A 股全市场日 K 线和基本面本地数据。

## 初始化

首次拉取项目后执行：

```bash
git submodule update --init --recursive
```

submodule 路径：

```text
vendor/stock-data-store
```

## 数据目录

默认数据写入当前项目的 `data/stock_data`：

```text
data/stock_data/
  daily/
    000001.SZ.csv
    600519.SH.csv
  fundamental/
    daily_basic/
    fina_indicator/
    income/
  meta/
    stock_basic.csv
    daily_last_ts/
    daily_update_progress.json
    fundamental_update_progress.json
```

`data/` 已在 `.gitignore` 中排除，不进入代码仓库。

## 更新命令

更新全市场前复权日 K：

```bash
.venv/bin/python -m src.aiagents_stock.cli.update_market_data --daily --refresh-stock-basic
```

更新基本面数据：

```bash
.venv/bin/python -m src.aiagents_stock.cli.update_market_data --fundamental --datasets daily_basic,fina_indicator,income
```

日 K 和基本面一起更新：

```bash
.venv/bin/python -m src.aiagents_stock.cli.update_market_data --all --refresh-stock-basic
```

小范围测试：

```bash
.venv/bin/python -m src.aiagents_stock.cli.update_market_data --daily --symbols 000001.SZ --force
.venv/bin/python -m src.aiagents_stock.cli.update_market_data --fundamental --symbols 000001.SZ --datasets daily_basic --force
```

## 除权刷新机制

日线默认使用 `qfq` 前复权数据。增量更新时会重新拉取本地最后一个交易日并比较远端前复权 `close` 与本地 `close`：

- 偏差未超过阈值：只追加新交易日。
- 偏差超过阈值：判定该股票发生前复权漂移，自动从上市日或默认起始日全量刷新该股票历史日线。

默认阈值：

```text
STOCK_DATA_ADJUSTMENT_PRICE_TOLERANCE=0.001
```

## 配置项

常用环境变量：

```text
TUSHARE_TOKEN=...
STOCK_DATA_ADJ=qfq
STOCK_DATA_MAX_WORKERS=4
STOCK_DATA_REQUESTS_PER_SECOND=4.0
STOCK_DATA_UNIVERSE_INCLUDE_ST=false
STOCK_DATA_UNIVERSE_MIN_LIST_DAYS=120
STOCK_DATA_ADJUSTMENT_DETECT_ENABLED=true
STOCK_DATA_ADJUSTMENT_PRICE_TOLERANCE=0.001
```

## 读取链路

`DataSourceManager.get_stock_hist_data()` 现在优先读取 `data/stock_data/daily/{ts_code}.csv`。本地数据不存在时，才回退到 Tushare/AKShare 的按需数据源。

这意味着后续选股应优先基于本地全市场数据仓库做批量计算，避免在选股过程中逐只股票请求远端接口。
