# 外部数据源总览

本文档整理项目运行时实际使用的外部数据源、第三方接口和外部通道。范围包括行情、财务、资讯、宏观、AI、通知和交易接口；不包含仅出现在部署文档中的包镜像、Docker 下载地址、B 站/图片等静态说明链接。

## 快速总览

|---|---|---|---|---|

## 行情、财务与市场数据

### AKShare

AKShare 是项目覆盖面最广的数据入口，代码里直接调用其 Python API，底层数据源由 AKShare 对接东方财富、新浪、同花顺、百度、交易所等公开数据源。

项目中主要使用的数据类型：

| 数据类型 | 典型接口 | 使用场景 |
|---|---|---|
| A股日线/分钟线 | `stock_zh_a_hist`、`stock_zh_a_hist_min_em` | 个股分析、智能盯盘、技术指标、价值策略 RSI |
| A股实时行情/基本信息 | `stock_zh_a_spot_em`、`stock_individual_info_em` | 个股信息、实时价格、换手率、市值、行业 |
| A股估值 | `stock_zh_valuation_baidu` | PE/PB 补充 |
| 港股行情/财务 | `stock_hk_spot_em`、`stock_hk_hist`、`stock_hk_financial_indicator_em` | 港股分析 |
| 财务报表/指标 | `stock_financial_report_sina`、`stock_financial_abstract_ths`、`stock_financial_abstract` | 季报、三表、财务指标 |
| 个股资金流 | `stock_individual_fund_flow`、`stock_individual_fund_flow_rank` | 资金面分析、智能盯盘主力资金 |
| 板块/概念/行业 | `stock_board_industry_name_em`、`stock_board_concept_name_em`、`stock_sector_fund_flow_rank` | 智策板块分析 |
| 北向资金 | `stock_hsgt_fund_flow_summary_em` | 智策北向资金备用源 |
| 指数 | `stock_zh_index_daily`、`stock_zh_index_spot_em` | 宏观、市场情绪、板块策略 |
| 涨跌停/两融 | `stock_zt_pool_em`、`stock_zt_pool_dtgc_em`、`stock_margin_*` | 市场情绪和风险分析 |
| 财经新闻 | `stock_news_em`、`stock_news_sina`、`stock_news_cls`、`stock_info_global_em` | 个股新闻、财联社电报、全球财经新闻 |
| 宏观/商品 | `macro_china_*`、`futures_main_sina` | 宏观周期、商品价格 |

关键调用位置：
- `src/aiagents_stock/integrations/market_data/providers.py`：A股历史、基本信息、实时行情、财务数据，失败后切换 Tushare。
- `src/aiagents_stock/features/stock_analysis/data.py`：A股/港股/美股主数据入口。
- `src/aiagents_stock/features/stock_analysis/fund_flow.py`：个股资金流，失败后切换 Tushare。
- `src/aiagents_stock/features/stock_analysis/quarterly_report_data.py`：最近 8 期季报。
- `src/aiagents_stock/features/stock_analysis/market_sentiment_data.py`：ARBR、换手率、大盘情绪、涨跌停、两融。
- `src/aiagents_stock/features/sector_strategy/data.py`：行业/概念板块、市场概况、北向资金、财经新闻。
- `src/aiagents_stock/features/macro_cycle/data.py`：宏观周期、利率、商品、地产、全球新闻。
- `src/aiagents_stock/features/macro_analysis/data.py`：市场指数、宏观新闻、候选股票快照。

### Tushare Pro

Tushare 是项目的主要备用数据源，需要配置 `TUSHARE_TOKEN`。当前逻辑通常是 AKShare 失败后再调用 Tushare，少数场景会优先使用 Tushare。

主要接口：
- `daily`：日线行情、实时行情兜底。
- `daily_basic`：PE、PB、换手率、市值等。
- `stock_basic`：股票名称、行业、市场、上市日期。
- `income`、`balancesheet`、`cashflow`：财务三表备用源。
- `moneyflow`：个股资金流备用源。
- `index_daily`：指数数据备用源。
- `moneyflow_hsgt`：智策北向资金优先源。

使用约束：
- 需要 token 和对应积分/权限。
- 部分接口有频率限制和积分门槛，代码中只做降级和日志提示，不做集中限流。

### TDX API

TDX API 是本地或局域网部署的行情服务，优先服务智能盯盘和实时监测。项目默认地址在不同文件/文档中存在历史差异，代码层当前常见默认值包括 `http://192.168.1.222:8181` 和 `http://127.0.0.1:5000`，实际以 `.env` 的 `TDX_BASE_URL` 为准。

主要接口：
- `GET /api/quote?code=000001`：实时行情。
- `GET /api/search?keyword=000001`：股票名称搜索。
- `GET /api/kline?code=000001&type=day`：K 线。
- `GET /api/health`：测试脚本健康检查。

调用位置：
- `src/aiagents_stock/features/smart_monitor/tdx_data.py`
- `src/aiagents_stock/features/realtime_monitor/service.py`
- `src/aiagents_stock/features/selectors/low_price_bull/service.py`
- `test_tdx_api.py`

降级关系：
- 智能盯盘：`TDX -> AKShare -> Tushare`。
- TDX 未启用或连接失败时，不阻断整体分析，会尝试后续数据源。

### yfinance / Yahoo Finance

yfinance 用于海外市场和港股兜底：
- 美股：行情、历史 K 线、资产负债表、利润表、现金流、估值与财务比率。
- 港股：当 AKShare/东方财富数据不可用时，用 `0700.HK` 这类 Yahoo 格式做行情和历史数据兜底。

调用位置：`src/aiagents_stock/features/stock_analysis/data.py`。

### pywencai / 问财

pywencai 用于自然语言查询，主要覆盖 AKShare 不容易直接表达的选股条件、公告和风险事件。

使用场景：
- 主力选股：区间主力净流入、市值区间、行业、财务评分等。
- 低价擒牛：低价、高净利增长、非 ST/非科创/非创业板。
- 小市值策略：总市值、营收增长、净利增长。
- 净利增长策略：深圳 A 股、净利润增长、成交额排序。
- 低估值价值策略：PE、PB、股息率、资产负债率。
- 新闻公告：`股票代码新闻`、`股票代码公告`。
- 风险事件：限售解禁、大股东减持、近期重要事件。

调用位置：
- `src/aiagents_stock/features/selectors/main_force/selector.py`
- `src/aiagents_stock/features/selectors/low_price_bull/selector.py`
- `src/aiagents_stock/features/selectors/small_cap/selector.py`
- `src/aiagents_stock/features/selectors/profit_growth/selector.py`
- `src/aiagents_stock/features/selectors/value_stock/selector.py`
- `src/aiagents_stock/features/stock_analysis/news_announcement_data.py`
- `src/aiagents_stock/features/stock_analysis/risk_data_fetcher.py`

## 资讯、龙虎榜与宏观数据

### 新闻热榜 API

新闻流量监测使用外部热榜聚合 API：

```text
https://orz.ai/api/v1/dailynews/?platform=<platform>
```

代码中保留了旧地址注释：

```text
https://newsapi.ws4.cn/api/v1/dailynews/
```

当前启用地址是 `orz.ai`。支持平台包括微博、抖音、知乎、B 站、小红书、快手、贴吧、微信、百度、今日头条、腾讯网、网易、凤凰、新浪、东方财富、雪球、财联社、华尔街见闻、36 氪、少数派、掘金等。

调用位置：`src/aiagents_stock/features/news_flow/data.py`。

### 龙虎榜 API

智瞰龙虎使用外部龙虎榜 API：

```text
http://lhb-api.ws4.cn/v1/youzi/all?date=YYYY-MM-DD
```

代码中还保留两个备选地址注释：

```text
https://api-lhb.zhongdu.net
https://www.stockapi.com.cn/v1
```

当前默认无需额外配置，但 `LonghubangDataFetcher` 构造函数保留 `api_key` 参数。代码内请求间隔为 `0.025` 秒，对应文档里提到的 40 次/秒限制。

调用位置：`src/aiagents_stock/features/longhubang/data.py`。

### 国家统计局接口

宏观分析板块直接请求国家统计局官方接口：

```text
https://data.stats.gov.cn/easyquery.htm
```

主要指标：
- GDP 当季同比、GDP 环比。
- 规上工业增加值同比。
- CPI、PPI。
- 制造业 PMI、非制造业商务活动指数、综合 PMI。
- M2 同比。
- 社零累计同比、固定资产投资累计同比、房地产开发投资累计同比。
- 全国城镇调查失业率。

调用位置：`src/aiagents_stock/features/macro_analysis/data.py`。

### AKShare 宏观与全球新闻

宏观周期板块通过 AKShare 获取：
- 中国 GDP、CPI、PPI、PMI、M2、LPR、房地产等宏观数据。
- 黄金、原油、铜等主力期货。
- 全球财经新闻。

调用位置：`src/aiagents_stock/features/macro_cycle/data.py`。

## AI、交易和通知通道

### OpenAI 兼容大模型 API

项目使用 OpenAI Python SDK 或直接 HTTP 调用 OpenAI 兼容接口，默认配置指向 DeepSeek：

```env
DEEPSEEK_API_KEY=...
DEEPSEEK_BASE_URL=https://api.deepseek.com/v1
DEFAULT_MODEL_NAME=deepseek-chat
```

代码和 UI 已支持替换为其他 OpenAI 兼容服务，例如硅基流动、火山方舟、阿里 DashScope 兼容模式等。实际服务由 `DEEPSEEK_BASE_URL` 和 `DEFAULT_MODEL_NAME` 决定。

调用位置：
- `src/aiagents_stock/integrations/llm/client.py`：统一 `DeepSeekClient`。
- `src/aiagents_stock/features/smart_monitor/agents.py`：直接 `POST {base_url}/chat/completions`。
- 各智能体模块通过 `DeepSeekClient` 或功能引擎间接调用。

### MiniQMT / xtquant

MiniQMT 是交易终端接口，不是行情备份源。项目通过 `xtquant.xttrader`、`xtquant.xtdata` 对接账户、持仓、买卖、撤单。

配置项：

```env
MINIQMT_ENABLED=true
MINIQMT_ACCOUNT_ID=...
MINIQMT_HOST=127.0.0.1
MINIQMT_PORT=58610
```

调用位置：
- `src/aiagents_stock/features/smart_monitor/qmt.py`：智能盯盘实盘/模拟交易适配器。
- `src/aiagents_stock/integrations/trading/miniqmt.py`：通用 MiniQMT 接口预留实现。

风险：启用后涉及真实账户和资金，必须先确认交易账户、权限、交易时间和自动策略参数。

### SMTP 邮件

邮件通知使用标准 SMTP，服务商由环境变量决定，例如 QQ、163、Gmail。

配置项：

```env
EMAIL_ENABLED=true
SMTP_SERVER=smtp.qq.com
SMTP_PORT=587
EMAIL_FROM=...
EMAIL_PASSWORD=...
EMAIL_TO=...
```

调用位置：
- `src/aiagents_stock/integrations/notification/service.py`
- `src/aiagents_stock/features/sector_strategy/scheduler.py`

说明：`env_example.txt` 中存在 `EMAIL_HOST`、`EMAIL_USER`、`EMAIL_RECIPIENT` 这类旧命名示例，但当前通知服务代码读取的是 `SMTP_SERVER`、`EMAIL_FROM`、`EMAIL_TO`。

### Webhook 通知

Webhook 用于群机器人通知。当前代码支持：
- 钉钉：`WEBHOOK_TYPE=dingtalk`
- 飞书：`WEBHOOK_TYPE=feishu`

配置项：

```env
WEBHOOK_ENABLED=true
WEBHOOK_TYPE=dingtalk
WEBHOOK_URL=https://oapi.dingtalk.com/robot/send?access_token=...
WEBHOOK_KEYWORD=股票
```

调用位置：
- `src/aiagents_stock/integrations/notification/service.py`
- `src/aiagents_stock/features/sector_strategy/scheduler.py`
- 低价擒牛/小市值/净利增长 UI 中也有直接 Webhook 测试或发送逻辑。

说明：`env_example.txt` 里有企业微信 `WECOM_WEBHOOK` 示例，但当前 `NotificationService` 未发现企业微信分支，运行时以 `WEBHOOK_TYPE` 的钉钉/飞书实现为准。

## 风险与维护要点

1. 外部金融数据源大多不是强 SLA 服务，字段名、限频、可用性可能变化；新增功能应复用现有数据源管理器或显式写出降级策略。
2. AKShare 底层来源分散，文档中应优先记录“项目调用的 AKShare 接口”和“业务用途”，不要假设底层网页字段长期稳定。
3. Tushare 需要 token、积分和接口权限；生产自动任务建议配置 `TUSHARE_TOKEN` 作为兜底。
4. TDX 和 MiniQMT 属于本地/局域网服务，部署文档里的端口示例不完全一致，实际运行以 `.env` 为准。
5. `env_example.txt` 与当前配置管理代码存在旧字段差异，新增配置说明应以 `src/aiagents_stock/core/config.py`、`src/aiagents_stock/app/pages/settings.py`、`src/aiagents_stock/integrations/notification/service.py` 为准。
6. Webhook URL、邮箱授权码、Tushare token、模型 API key 都属于敏感信息，不应提交到版本库。
