# 智策板块资金流功能改造方案

## 目标

在“智策板块”页面新增一个独立子功能“板块资金流”，用于查看指定时间范围内申万行业板块的资金净流入情况，并查看板块内部资金净流入排名靠前的个股。

本功能暂不接入 AI 分析，不调用现有 4 个 AI 分析师，也不参与首席策略综合研判。

## 边界

- 不影响现有“智策板块”综合研判入口。
- 不改现有 `sector_analysis_reports` AI 报告表。
- 不使用高积分板块资金接口。
- 数据口径暂时只使用 Tushare 2000 积分可用接口。
- 板块范围暂时只支持申万行业分类，不支持同花顺概念、东方财富概念等高积分板块体系。

## 数据口径

使用 Tushare 接口：

- `index_classify`：申万行业分类，2000 积分可用。
- `index_member_all`：申万行业成分股，2000 积分可用。
- `moneyflow`：个股资金流向，2000 积分可用。
- `trade_cal`：交易日历，用于解析时间窗口。

板块资金流不是直接读取 Tushare 官方板块资金流接口，而是按申万行业成分股的 `moneyflow` 明细聚合：

- 板块净流入 = 成分股 `net_mf_amount` 汇总。
- 主力净流入 = 大单净流入 + 特大单净流入。
- 特大单净流入 = `buy_elg_amount - sell_elg_amount`。
- 大单净流入 = `buy_lg_amount - sell_lg_amount`。
- 中单净流入 = `buy_md_amount - sell_md_amount`。
- 小单净流入 = `buy_sm_amount - sell_sm_amount`。

金额单位沿用 Tushare `moneyflow`：万元。

## 时间范围

前端支持：

- 最近一周：`1w`
- 最近两周：`2w`
- 最近一月：`1m`
- 最近两月：`2m`
- 最近三月：`3m`
- 某天：`day`
- 自定义区间：`custom`

后端统一解析为 `start_date`、`end_date` 和实际交易日列表。

## 后端设计

新增模块：

- `src/aiagents_stock/features/sector_strategy/fund_flow.py`
- `src/aiagents_stock/features/sector_strategy/fund_flow_repository.py`

新增 API：

- `POST /api/sector-strategy/fund-flow/run`
- `GET /api/sector-strategy/fund-flow/reports`

## 独立数据库表

数据库文件：`sector_fund_flow.db`

新增表：

- `sector_member_cache`
  - 缓存申万行业成分关系。
- `sector_flow_daily`
  - 按交易日、板块层级保存板块资金流聚合结果。
- `sector_flow_stock_daily`
  - 按交易日、板块层级保存板块内个股资金流结果。
- `sector_fund_flow_reports`
  - 保存一次板块资金流分析报告及运行参数。

这些表独立于现有智策 AI 报告表，避免影响旧功能。

## 前端设计

将 `/sector-strategy` 页面改成专用页面，保留原综合研判入口，并新增标签：

- 综合研判：沿用现有 `/sector-strategy/run` 和历史报告。
- 板块资金流：使用新的 `/sector-strategy/fund-flow/run` 和独立历史报告。

板块资金流页面包含：

- 时间范围选择。
- 某天/自定义日期输入。
- 申万行业层级选择：一级、二级、三级。
- TOP 板块数量。
- 板块内 TOP 个股数量。
- 运行按钮。
- 板块净流入排行。
- 板块净流出排行。
- 点击板块展开查看个股资金流 TOP N。

## 测试策略

- 测试时间窗口解析。
- 使用 fake Tushare API 测试申万成分与 `moneyflow` 聚合。
- 测试缓存命中时不重复请求 `moneyflow`。
- 测试报告独立入库和读取。
- 跑现有全量测试，确认现有智策板块综合研判不受影响。
