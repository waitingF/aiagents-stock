# 股票池功能改造方案

## 目标

新增“股票池”作为投资管理的统一底层。旧“持仓分析”不再维护独立底层，直接切换为 `持仓` 股票池的专用视图。

## 默认股票池

系统默认创建两个股票池：

- `持仓`：系统池，不可删除。由现有“持仓分析”中的股票和历史迁移而来。
- `自选`：系统池，不可删除。默认空池，用于观察标的。

用户可以新增普通股票池，如 `短线观察`、`低估值`、`财报季重点`。

## 核心规则

- 同一股票可以存在于多个股票池。
- 同一股票在同一个股票池内不能重复。
- `stock_pool_items.created_at` 记录股票加入该池的时间。
- 当前阶段不增加操作流水表。
- 选择多个股票池分析时，同一股票只跑一次 AI 分析。
- 分析结果按 `pool_id + pool_item_id` 分别写回多个股票池。
- 每个股票池拥有独立分析历史和每日复盘。
- 旧 `portfolio_*` 表只作为迁移来源，业务不再写入。

## 初始化与迁移

应用启动时执行幂等初始化：

1. 创建 `持仓` 池：`type='holding'`、`is_system=1`。
2. 创建 `自选` 池：`type='watchlist'`、`is_system=1`。
3. 从旧 `portfolio_stocks` 迁移到 `持仓` 池。
4. 从旧 `portfolio_analysis_history` 迁移到 `stock_pool_analysis_history`。
5. 迁移过程可重复执行，不重复插入。
6. 迁移后业务代码不再读写旧 `portfolio_*` 表。

## 股票池分析流程

选择多个股票池时，构建目标映射：

```python
analysis_targets = {
    "600519.SH": [
        {"pool_id": 1, "pool_item_id": 10},
        {"pool_id": 2, "pool_item_id": 35},
    ],
}
```

执行规则：

1. 每个股票代码只调用一次 `analyze_single_stock_for_batch`。
2. 保存时按 `pool_id + pool_item_id` 分别写入历史。
3. 复盘基于该股票在对应股票池中的上一条历史独立生成。

## 第一阶段范围

第一阶段直接完成统一底层：

1. 新建 `stock_pool` 模块和表。
2. 默认创建 `持仓`、`自选`。
3. 迁移旧持仓股票和历史到 `持仓` 池。
4. `持仓分析` 切换为 `持仓` 股票池专用视图。
5. 新增 `股票池` 全量管理页面。
6. 实现股票池 CRUD、成员 CRUD、批量导入。
7. 实现选池、选股票、顺序/并行分析。
8. 实现同股跨池只分析一次、多池分别写历史。
9. 实现每日复盘和历史查看。
10. 接入实时监测同步。
11. 增加基础测试。
